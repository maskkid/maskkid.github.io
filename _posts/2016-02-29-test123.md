**碎碎念**

        希望尽快度过这段时期，踏踏实实的规划，认认真真的学习，快快乐乐的生活：）

        在计算机方向这么多年的经历，一直算是爱好驱动。在几年的时间里，对很多东西都感兴趣，包括各种语言，各种概念，各种物联方向、图像识别、语音、发声、学习算法、数据分析，各种框架。。。

        对技术的迷失断断续续有一年半了，好处是知识面扩展不少，基本的计算机应用方向的要求都会有解决方案；坏处，深度不够，本身竞争力太小。

        从语言的角度，对c++、c#、java、javascript无所适从；从方向上，对2D游戏开发、3D游戏开发、物联网、arvr、移动开发、前端架构选择障碍。

**列一下目前技能和选择障碍内容：**

* 当前技能
    * 前端技能（html/css/javascript）：基本熟悉页面构建的知识，了解js的对象基础和dom操作，对jquery、zepto、understore、angularjs、backbone、reactjs、ionic等等前端框架和库有所认识。在前端架构上，目前问题不大；
    * php/mysql：基本熟悉php和mysql的web解决方案，对tk、ci、yii简单了解，对其他业内的框架方案选择基本了解；
    * 服务器环境，搭建没问题，基本优化知道；
    * java/android能解决日常工作的工具需求；
    * python系统学习过，水平中等；
    * nodejs，10年前后就搞过自己的server端，许久不用，用即能用。

* 学习选择（以下全部入门，深入方向迷失）
   * 游戏开发。难度在图形、算法学，深入需要很长时间。好的一方面，兴趣浓厚。
        * 2D选择 入门和使用简单，兴趣点小；
        * 3D 非常喜欢，尤其是针对ARVR方向。但是深入的话，对建模、美术设计要求很高，而这些方面也不是一朝一夕能学出来的。
   * 深入语言
        * c++ 深入难度大，个人时间少，不能短期深入。另外对长久发展持怀疑态度；
        * java 方向混乱，相比python，解决普通问题非常啰嗦，浪费时间。但是在第三方支持上很好；
        * c# 跟java类似，但是平台方面比较担心。目前主要是为了u3d而学习，本心也想学好；
        * python 工作选择面太窄，入门简单，几年后竞争压力很大；
        * nodejs 同上。

* 必学知识
    * 数据结构/算法/计算机原理
    * 数据分析/统计学
    * 设计模式

**临时选择**

            鉴于工作能力、个人时间两方面考虑，应该重点放到前端开发深入、php/mysql深入、2D游戏开发三方面。前两点基本在工作中搞定，后面2D开发也如此。

            这么选择的坏处是，这些方面的技术含量和未来发展不是很看好，伴随年龄的增加、精力体力的下降，会一直在学习中，健康和工作的反比系数会越来越大，风险太高。
