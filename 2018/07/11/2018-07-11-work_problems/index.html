前段时间总结的一些开发问题

Posted on 2018-07-11 Edited on 2021-08-13 In work

这是前段时间在一套不认可的前端方案下，后端几十号人完全没有规范的情况下，总结了一个问题和建议的文档。

前端部分

  1. 框架底层库不稳定,尤其是自建库部分,目前经常需要变动,导致页面来回改动. 建议: 封一层,内部完全只用antd.
  2. 目前写一个页面创建需要做以下几件事:
    • 添加路由配置(改写loadfile和index两个文件) 2
    • 分析页面数据结构,创建数据结构体(前端叫法 interface) 1
    • 分析页面,隔离编写组件/容器/页面三部分 2-3
    • 编写redux以及数据服务部分,涉及以下几个目录和文件的创建:
      • actions/模块名/页面文件 1
      • reducers/模块名/页面文件,创建完成后,改写同目录下index文件配置,然后再改写reducers目录下的index文件 3
      • api 1
      • mock编写(mock具体文件编写/db.js/routes.js) 3

以上过程,纯手工编写,非常繁琐,建议:尽量把路由/api/mock部分提前配置,自动生成(依赖后端同事,尽快出全部的接口定义)

接口部分

  1. 文档不统一,生成的文档每个模块接口都不一样
  2. 没有接口规范,可以参考币别管理接口
  3. 接口容错和提醒很暴力,最起码应该区分一下业务错误是数据格式问题,还是数据项类型问题
  4. 接口交付前,一定要测试,并且有测试通过的原始数据格式和测试结果记录

接口要么没有定义;有定义的,没字段注释;有字段注释的,调不通;调通了,又要改

对接例子eq.
企业信息接口 查询/添加 一周
摊销 一周
财税设置 一周
币别管理 四个接口 一下午
