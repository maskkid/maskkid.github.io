个人OCR学习使用 

        几年前接触ocr的东西，没有正式使用，只训练了几组中文字库就略过了。这次朋友正好用到，重温一边，软件和过程都整理文档方便备用。真正要做到ocr高识别，必须精通算法和神经网络。此篇为外行浅入，玩玩总可以吧。
        感谢tesseract的开源，感谢google的各种支持，感谢网友们无私的文档奉献。

语言、编译环境

  • java
  • python（2.7） pip PIL

软件介绍

  • tesseract 30年前HP的开源的作品，后来google接管了。ocr不二的推荐。

  • jTessBoxEditor 库文件编辑工具

  • css222.py 图片二值化脚本，针对rgb值，还能再优化。

  • ttf2box.bat ttf文件调用tesseract命令

  • create.bat 最终生成训练库文件的脚本

  • tesseract

    • 下载地址：https://code.google.com/p/tesseract-ocr/downloads/list

  • jTessBoxEditor 训练工具

    • http://sourceforge.net/projects/vietocr/files/jTessBoxEditor/

操作流程

  1. 使用jTessBoxEditor拼合库文件， 格式ttf
  2. 通过css222.py 进行二值化
  3. 通过ttf2box.bat 生成box文件
  4. 创建必须文件，参考文档。 font啊，charset啊等等
  5. 通过create.bat生成最终库文件 (比如叫 abc.库文件后缀)
  6. 把库文件导入tesseract的库目录
  7. 识别：通过tesseract 目标文件 结果文件 -l abc

ThinkPHP 源码分析 

        ThinkPHP 3 
        利用大概一天的时间，把ThinkPHP的源码看了一些，主要是对Tk框架设计进行分析学习，在php框架的设计上吸收些知识。
        主要从目录结构，框架的流程，以及定制化的设计上进行记录备用。

  • 目录结构[官方文档部分内容和文件结构和3.*版有偏差]

    • 框架基本结构

        ├─ThinkPHP.php     框架入口文件
   ├─Common 框架公共文件
   ├─Conf 框架配置文件
   ├─Extend 框架扩展目录
   ├─Lang 核心语言包目录
   ├─Lib 核心类库目录
   │  ├─Behavior 核心行为类库
   │  ├─Core 核心基类库
   │  ├─Driver 内置驱动
   │  │  ├─Cache 内置缓存驱动
   │  │  ├─Db 内置数据库驱动
   │  │  ├─TagLib 内置标签驱动
   │  │  └─Template 内置模板引擎驱动
   │  └─Template 内置模板引擎
   └─Tpl 系统模板目录

- 应用的基本结构

        ├─index.php     项目入口文件
         ├─Common 项目公共文件目录
         ├─Conf 项目配置目录
         ├─Lang 项目语言目录
         ├─Lib 项目类库目录
         │  ├─Action Action类库目录
         │  ├─Behavior 行为类库目录
         │  ├─Model 模型类库目录
         │  └─Widget Widget类库目录
         ├─Runtime 项目运行时目录
         │  ├─Cache 模板缓存目录
         │  ├─Data 数据缓存目录
         │  ├─Logs 日志文件目录
         │  └─Temp 临时缓存目录
         └─Tpl 项目模板目录

  • 主流程

    • index.php - 入口文件
      • ThinkPHP.php [Think::start()] - 主类，define主要变量
        • ThinkPHP.class.php [App::run()] - 核心类库，注册autoload/错误处理/加载系统和应用配置文件
          • App.class.php [ run -> init[Dispatch::dispatch 解析url] + exec ] - 应用主类，解析MVC，解析过滤参数，反射调用对应的action

  • 主要文件说明[待续]

    • functions.php
    • Model.class.php
    • Controller.class.php
    • Action.class.php
    • Cache

  • 定制化

    • 修改模板调用方式。view层只原生php代码。添加自定义模板方法,模仿目前公司框架的形式。 functions.php添加 simotpl(),simoview()两个方法

ReactJs 总结

简介

Reactjs是facebook开源的一个前端框架，web方向仅处理View层。更大的野心在多端处理上，实现跨端的虚拟标识框架。相比目前我认识的框架，要简单/轻量的多。

应用

个人理解，主要解决同样的代码/结构/动作只写一次，复用性，模块化非常方便。可以父子包含/并列组合。配合JSX，清新干爽，你说像什么就是什么：）

干货

  • 组件创建，creatCLass
  • 静态数据传递/获取， 属性/props
  • 动态数据传递/获取/设置， state/setSstate，发生变化后，会自动调用render
  • 虚拟节点子项获取，ref/refs.ref
  • 其他操作同js
  • MC层可以组合backbone/spine等框架使用

学习方向迷失mark新路

碎碎念

    希望尽快度过这段时期，踏踏实实的规划，认认真真的学习，快快乐乐的生活：）

    在计算机方向这么多年的经历，一直算是爱好驱动。在几年的时间里，对很多东西都感兴趣，包括各种语言，各种概念，各种物联方向、图像识别、语音、发声、学习算法、数据分析，各种框架。。。

    对技术的迷失断断续续有一年半了，好处是知识面扩展不少，基本的计算机应用方向的要求都会有解决方案；坏处，深度不够，本身竞争力太小。

    从语言的角度，对c++、c#、java、javascript无所适从；从方向上，对2D游戏开发、3D游戏开发、物联网、arvr、移动开发、前端架构选择障碍。

列一下目前技能和选择障碍内容：

  • 当前技能

    • 前端技能（html/css/javascript）：基本熟悉页面构建的知识，了解js的对象基础和dom操作，对jquery、zepto、understore、angularjs、backbone、reactjs、ionic等等前端框架和库有所认识。在前端架构上，目前问题不大；
    • php/mysql：基本熟悉php和mysql的web解决方案，对tk、ci、yii简单了解，对其他业内的框架方案选择基本了解；
    • 服务器环境，搭建没问题，基本优化知道；
    • java/android能解决日常工作的工具需求；
    • python系统学习过，水平中等；
    • nodejs，10年前后就搞过自己的server端，许久不用，用即能用。

  • 学习选择（以下全部入门，深入方向迷失）

    • 游戏开发。难度在图形、算法学，深入需要很长时间。好的一方面，兴趣浓厚。
      • 2D选择 入门和使用简单，兴趣点小；
      • 3D 非常喜欢，尤其是针对ARVR方向。但是深入的话，对建模、美术设计要求很高，而这些方面也不是一朝一夕能学出来的。
    • 深入语言
      • c++ 深入难度大，个人时间少，不能短期深入。另外对长久发展持怀疑态度；
      • java 方向混乱，相比python，解决普通问题非常啰嗦，浪费时间。但是在第三方支持上很好；
      • c# 跟java类似，但是平台方面比较担心。目前主要是为了u3d而学习，本心也想学好；
      • python 工作选择面太窄，入门简单，几年后竞争压力很大；
      • nodejs 同上。

  • 必学知识

    • 数据结构/算法/计算机原理
    • 数据分析/统计学
    • 设计模式

临时选择

        鉴于工作能力、个人时间两方面考虑，应该重点放到前端开发深入、php/mysql深入、2D游戏开发三方面。前两点基本在工作中搞定，后面2D开发也如此。

        这么选择的坏处是，这些方面的技术含量和未来发展不是很看好，伴随年龄的增加、精力体力的下降，会一直在学习中，健康和工作的反比系数会越来越大，风险太高。
